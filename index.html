Cisco CLI Notes

Basic Mode
Basic Router/Switch Configuration
enable password [text]               # Privileged EXEC unencrypted password
enable secret [text]                 # Privileged EXEC secret
# You can NOT have both an enable password and enable secret set at the same time!
line console 0                       # Enters console line
password [text]                      # Sets console password
service password-encryption          # Encrypts all plaintext passwords on the device
login                                # Requires login on console line
hostname [text]                      # Sets the hostname
banner motd [symbol][text][symbol]   # Message displayed upon login
copy run start | write mem           # Saves running configuration
Example Commands
R1(config)# enable secret C1sc0R0cks
R1(config)# line console 0
R1(config-line)# password Cisco
R1(config-line)# login
R1(config-line)# exit
R1(config)# hostname Router1
Router1(config)# service password-encryption
Router1(config)# banner motd #Authorized Access Only!#
Router1(config)# write mem
Interface Addressing
interface [Interfacename]        # Enters interface configuration mode for that interface
ip address X.X.X.X ne.tm.as.k    # Sets the interface IP address
no shutdown                      # Activates the interface
Example Commands
R1(config)#interface g0/0/0
R1(config-if)#ip address 192.168.1.1 255.255.255.0
R1(config-if)#no shutdown
Enabling SSH
ip domain-name [domain]                # Sets the default domain name
crypto key generate rsa   (1024)       # Generates a new RSA key. Hostname cannot be default
username [text] password/secret [text] # Declares a new user with credentials
ip ssh version [num]                   # Sets the SSH Version
transport input ssh 		       # Enables SSH on the VTY lines
Example Commands
R1(config)#ip domain-name cisco.com
R1(config)#crypto key generate rsa
R1(config)#ip ssh version 2
R1(config)#username Admin secret hi
R1(config)#line vty 0 4
R1(config-line)#transport input ssh
R1(config-line)#login local

Advanced: Stronger Encrypted Passwords
enable secret 5/8/9 [Hash]                                # Sets the enable secret to a pre-generated hash
enable algorithm-type [Hashtype] secret [text]            # Derives the hashed enable secret from a plaintext
username [text] algorithm-type [Hashtype] secret [text]   # Makes a new user with a hashed password derived from a plaintext
Example Commands
enable secret 5 dfeaf10390e560aea745ccba53e044ed
enable algorithm-type md5 secret cisco
username Admin algorithm-type md5 secret cisco
